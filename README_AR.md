# 🏥 نظام ERP الطبي المتكامل - Medical ERP System

نظام احترافي وعالي الأداء لإدارة الموارد الطبية (ERP) مصمم للعيادات والمستشفيات الحديثة. يتميز النظام بواجهة مستخدم جميلة تدعم الوضع الليلي والنهاري، وتحديثات لحظية، وتحكم في الوصول متعدد الأدوار.

---

## 🚀 الوحدات والميزات الرئيسية

### 📊 مركز التحليلات
- **إحصائيات لحظية**: تتبع إيرادات اليوم، وعدد المرضى، والمواعيد المعلقة.
- **تمثيل البيانات**: مخططات تفاعلية لاتجاهات الإيرادات وأنواع الزيارات.

### 💊 الصيدلية والمخزون
- **قاعدة بيانات الأدوية**: إدارة كاملة (CRUD) للأدوية السريرية.
- **إدارة المخزون**: تنبيهات تلقائية لنقص المخزون وسجل للأدوية المنصرفة.
- **الفواتير التلقائية**: تكامل مع روشتات الأطباء لإصدار الفواتير بسلاسة.

### ⏳ إدارة الطابور
- **طابور ذكي**: تحكم لحظي في تدفق المرضى لموظفي الاستقبال والأطباء.
- **نداء المريض**: وظيفة "نداء التالي" مسموعة ومرئية.
- **📺 شاشة العرض العامة**: مسار مخصص بملء الشاشة (`/public-queue`) لشاشات التلفاز في غرف الانتظار.

### 🖨️ الطباعة الاحترافية
- **تنسيقات نظيفة**: قوالب مصممة خصيصاً للفواتير، الروشتات (Rx)، وتقارير المختبر.
- **طباعة صامتة**: محسنة لمحركات الطباعة في المتصفحات الحديثة.

### 🛡️ التحكم في الوصول على أساس الأدوار (RBAC)
- **مدير (Admin)**: وصول كامل للنظام، إدارة الموظفين، وسجلات المراجعة.
- **طبيب (Doctor)**: التركيز على تاريخ المريض، الروشتات الإلكترونية، ونتائج المختبر.
- **كاشير (كاشير)**: يتولى تسجيل المرضى، الجدولة، وتحصيل المدفوعات.

### 🌍 الموقع والثيم
- **عربي/إنجليزي بالكامل**: تبديل سلس مع ترجمات سريرية عالية الجودة.
- **واجهة مستخدم عصرية**: مبنية بمعايير Shadcn/UI، وتأثيرات Glassmorphism، وحركات Framer Motion.

---

## 🛠️ التقنيات المستخدمة

- **الواجهة الأمامية**: React (Vite), Tailwind CSS 4.0, Lucide Icons, Framer Motion, Axios.
- **الواجهة الخلفية**: Node.js, Express, Socket.io (Real-time).
- **قاعدة البيانات**: MongoDB (Mongoose).
- **الأمان**: JWT Authentication, Bcrypt Password Hashing, Audit Logging Middleware.

---

## ⚙️ التثبيت والإعداد

1. **نسخ المستودع (Clone)**
2. **تثبيت التبعيات**
   - الواجهة الخلفية: `npm install` داخل مجلد `/server`.
   - الواجهة الأمامية: `npm install` داخل مجلد `/client`.
3. **متغيرات البيئة (Environment Variables)**
   - أنشئ ملف `.env` في مجلد `/server` يحتوي على:
     - `MONGODB_URI`
     - `JWT_SECRET`
     - `PORT` (الافتراضي 5000)
4. **تشغيل التطبيق**
   - الواجهة الخلفية: `npm run dev`
   - الواجهة الأمامية: `npm run dev`
